[DISCOVERIES]
- 2026-04-06T16:40:00+0200 [CODE] Physically expanding `plugins=["cadd"]` into two active plugin entries would duplicate `cadd_raw`/`cadd_phred` in the structured schema and `CSQ`. The safe split was to keep `PluginKind::Cadd` logical and introduce separate physical source kinds only for discovery/build/lookup routing.
- 2026-04-06T20:30:00+0200 [CODE] `CsvReadOptions.comment(b'#')` was incompatible with plugin TSV headers such as `#CHROM`, `#chr`, and `#Chrom`: it silently dropped the real header row so the first data row became the inferred schema. Removing the comment marker fixed `AlphaMissense` and `dbNSFP` conversion.
- 2026-04-06T20:30:00+0200 [CODE] `dbNSFP` source headers are mixed-case and must be referenced as quoted identifiers in DataFusion SQL. Unquoted column names were normalized to lowercase and caused field-resolution failures even when the source TSV schema was otherwise correct.
- 2026-04-07T10:15:00+0200 [CODE] The `plugins_dir` runtime option never actually required a literal directory named `plugins`; it only needed a base path under which plugin parquet directories were discovered by name. That made the layout flattening low-risk once `vepyr.annotate()` and `build_plugin()` stopped prepending `plugins/`.
- 2026-04-07T11:25:00+0200 [CODE] Matching issue-style dbNSFP schema required changing more than aliases: transcript-scoped score columns such as `sift4g_score`, `metasvm_score`, and `metalr_score` must stay `Utf8` in parquet/runtime to preserve semicolon-separated multi-value payloads. Only genuinely scalar metrics like `revel_score`, `gerp_rs`, `phylop*`, `phastcons*`, and dbNSFP-provided `cadd_*` remain numeric.
- 2026-04-07T11:25:00+0200 [CODE] A shared `cadd` parquet cache made the old SNV-vs-indel runtime routing unnecessary. After merge, lookup can use one plugin index keyed by `(pos, ref, alt)` and preserve both SNV and indel matches without variant-type branching.
- 2026-04-09T13:35:00+02:00 [CODE] The existing Ensembl cache split is already the right template for plugin refactoring: `datafusion-bio-formats` owns raw-source `TableProvider` and export-query knowledge, while `datafusion-bio-functions` owns builder orchestration, parquet writing, Fjall building, and runtime lookup. Plugin work had drifted out of that shape because `plugin_cache_builder.rs` accumulated raw-source registration, sanitization, and source-specific SQL directly.
